Hierarchy-aware top bar

Root screens use title and contextual actions without Back. Nested screens use Back; after scroll they collapse to Back, a truncated title, and essential utilities. Recipe Detail adds identity and local modes only in its expanded state.

Edit Mode

Read current decisions
Approved · option B

Desktop: left section rail

One long editing page with wide sections, persistent section counts and validation state, and explicit global Save changes / Cancel.

Approved · option B

Mobile: compact section index

The rail becomes a sticky current-section control. It opens a navigational bottom sheet and closes immediately after a destination is selected.

Approved

Edit with Media or Import Info open

Full rail when the editor remains wide; compact selector beside a reflowing drawer; unchanged inert editor beneath a narrow-desktop overlay. Unsaved recipe edits are preserved.

Approved

Manage Media is Media’s Edit state

One panel handles capacity, upload, cover selection, and image deletion with its own draft. External links remain visible but read-only; View and Focus Media remain read-only.

Approved

Unit: fixed-dictionary autocomplete

Desktop expands the active row inline. Mobile exposes the same bounded, filterable choices inside the focused single-ingredient sheet.

Approved · option A

Mobile Ingredients: compact list + single-item editor

Rows remain easy to scan and reorder. The full summary button with a chevron opens one focused ingredient sheet; Done returns its sub-draft to the unsaved Recipe Edit draft.
