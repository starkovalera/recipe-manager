# UX Review — Feedback Refinement

Status: Proposed for approval  
Updated: 2026-07-22

## Scope

This iteration evaluates Default View, simplified Focus, and Import Info using realistic normal, flagged, dense, long, and mobile scenarios. It preserves the approved B2 metadata order and does not establish a high-fidelity visual language.

## Findings

- The compact review status preserves the recipe as the primary task and still gives flags a clear route into Import Info.
- Separating source identity from time and servings removes a false equivalence between provenance and cooking facts.
- Independent disclosure for Ingredients, Instructions, and Notes makes long recipes scannable without hiding whole content areas behind one global control.
- Shared View / Focus / Edit navigation removes the forced Focus-to-View detour before editing.
- One bulk `Mark all reviewed` action matches general, non-field-specific flags. Its confirmation correctly says that recipe content and resources will not change.
- Grouping derived resources under their primary resource makes cascade deletion understandable before the confirmation step. Individual derived removal remains available.

## Remaining decision

Approve or revise the proposed resource hierarchy after visual inspection. The data relationship is correct; labels, density, and removal affordances remain low fidelity.
