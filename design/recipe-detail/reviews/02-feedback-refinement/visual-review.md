# Visual Review — Feedback Refinement

Status: Proposed for approval  
Updated: 2026-07-22

## Header action comparison

### A — Under title

The horizontal cluster stays with the recipe identity, uses less vertical space, and makes View / Focus / Edit read as peer contexts. It leaves the cover as supporting media and keeps the review status directly below the task controls.

Risk: on narrower widths the cluster may wrap and needs careful high-fidelity spacing.

### B — Under cover

The vertical stack is easy to locate and visually separates actions from metadata. It can work when the cover acts as a persistent utility rail.

Risk: it gives a small image disproportionate structural weight, creates a tall control column, and separates actions from the title they affect. The page begins to resemble a sidebar layout.

## Recommendation

Choose **A — Under title**. It has the stronger task relationship and preserves the compact-header goal. Keep B only if future recipe-level tools justify a real persistent utility rail.

## Resource hierarchy

The single primary row plus indented connector is clearer and lighter than nested cards. Type labels, child count, status, and the protected-cover label are visible without duplicating recipe content. High fidelity should reduce line density and replace the provisional multiplication-sign remove glyph with an accessible icon and tooltip.
