# Reference Research Brief

## Goal

Find current, high-quality product patterns that can inform Recipe Manager without turning it into a copy of another product.

## Research lanes

Study at least one strong reference in each relevant lane:

1. productivity object/detail pages;
2. long-form reading and structured content;
3. content editing with many repeated rows;
4. metadata organization and large tag sets;
5. import provenance, source inspection, or review workspaces;
6. focused reading or execution modes;
7. media drawers and mobile sheets;
8. responsive dense-data interfaces.

## Evidence requirements

For each reference:

- use a current source;
- identify the exact screen or pattern;
- include a link or source citation;
- describe the interaction, not only the visual style;
- explain fit and non-fit for Recipe Manager;
- extract a principle rather than copying a layout.

## Comparison format

```text
Reference:
Pattern:
Useful because:
Risk:
Applicable decision:
```

## Avoid

- moodboards made only of colors and typography;
- relying only on recipe applications;
- copying a marketing website;
- citing a product name without studying a concrete screen;
- using screenshots as unquestionable answers;
- inventing current product behavior from memory.
